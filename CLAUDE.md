## Project Overview

Atari 800/800XL/65XE/130XE FPGA emulation on the Sipeed Tang Nano 20K (GW2AR-LV18QN88PC8/I7).
Adapted from [Atari800_MiSTer](https://github.com/MiSTer-devel/Atari800_MiSTer) for the Gowin toolchain.

## Build

```bash
QT_QPA_PLATFORM=offscreen /home/carlos/Documents/gowin/IDE/bin/gw_sh.sh build.tcl 2>&1 | tee build.log
```

Output bitstream: `impl/atari800_tn20k/impl/pnr/atari800_tn20k.fs`

Flash: use Gowin Programmer GUI or `programmer_cli --device GW2AR-LV18QN88PC8/I7`.

## Repository Layout

```
build.tcl                  # Gowin build script (add all source files here)
constraints/tang_nano_20k.cst  # Physical pin assignments (QFN88P numeric indices)
rtl/                       # Upstream Atari core VHDL (DO NOT modify)
src/                       # Tang Nano-specific SystemVerilog/Verilog/VHDL
```

Key files in `src/`:
- `tang_top.sv` — top-level; wires everything together
- `hdmi_audio_out.sv` — HDMI output: TMDS video + HDMI 1.3 audio data islands
- `terc4.sv` — TERC4 encoder (4-bit → 10-bit, used by data island channels)
- `tmds_encoder.sv` — TMDS 8b10b encoder (video channels)
- `sigma_delta_dac.sv` — first-order sigma-delta DAC for GPIO PDM audio
- `rpll_135m.v` / `rpll_12m.v` — PLLs: 27→135 MHz (HDMI), 27→12 MHz (USB)
- `sd_rom_loader.sv` / `fat_reader.sv` / `sd_card.sv` / `spi_master.sv` — SD FAT32 ROM loader
- `usb_hid_host.v` / `usb_to_atari800.sv` — USB HID keyboard

## Architecture

### Clock domains
- `clk_sys` = 27 MHz (pixel clock, Atari core, SDRAM adapter, audio tick)
- `clk_5x` = 135 MHz (OSER10 HDMI serialiser)
- `clk_usb` = 12 MHz (USB HID host)

### Reset hierarchy
- `hw_reset_n` = button AND both PLLs locked
- `core_reset_n` = hw_reset_n AND roms_loaded (Atari core held in reset until SD load completes)

### Memory
The GW2AR-18 has embedded 64 Mbit SDRAM accessed via the Gowin `SDRC_HS` IP.
`tang_top.sv` contains a simple request/ack adapter (SA_IDLE/SA_BUSY).
ROM loader has SDRAM priority until `roms_loaded`; then the Atari core takes over.

### HDMI audio (hdmi_audio_out.sv)
State machine per horizontal blanking: S_ACTIVE → S_CTRL → S_PRE(8) → S_LGB(2) → S_DATA(32) → S_TGB(2) → repeat up to 2 islands.
- First island after `vs` falling edge: ACR packet (N=6144, CTS=27000 → 48 kHz at 27 MHz)
- Subsequent islands: Audio Sample packets from an 8-entry FIFO
- 48 kHz tick generated by Bresenham accumulator (+=16, tick at 9000; exact 48000 Hz average)
- GPIO sigma-delta DAC on pins 33/34 is always active in parallel

### Upstream VHDL core (`rtl/`)
`atari800core_simple_sdram` is the top-level Atari core entity. Its key generics
set in `tang_top.sv`: `cycle_length=16`, `palette=1` (Altirra), ROMs/RAM in SDRAM.
Do not modify files under `rtl/` — sync from upstream Atari800_MiSTer instead.

## Adding Source Files

1. Create the file in `src/`
2. Add `add_file "$src_dir/<filename>"` to `build.tcl` in the appropriate stage section
3. Instantiate in `tang_top.sv`

## Pin Constraints

`constraints/tang_nano_20k.cst` uses numeric QFN88P pin indices.
Bank 7 (LPLL1/SDRAM area) is 1.8 V — do not assign user I/O there.
HDMI pins (25–32) are fixed to ELVDS_OBUF inside `hdmi_audio_out.sv`; no CST entry needed.

## Confirmed Hardware Pin Facts

- LEDs: pins **15–20** (active LOW). Pin 11 is NOT an LED — it is IOL29B adjacent to the 27 MHz clock input on pin 10.
- LED polarity confirmed on hardware: drive LOW = ON.

## Known Toolchain Quirks

- Use `gw_sh.sh` (not `gw_sh`) for headless builds
- `NL0002` "swept in optimizing" warnings are module flattening (logic preserved), not errors
- `PA1019` PLL VCO range warnings are pre-existing and harmless
- `SDRC_HS` config files must be copied to the Gowin IP data dir before synthesis (handled in `build.tcl`)
